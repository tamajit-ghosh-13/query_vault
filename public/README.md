# QueryVault — Mock REST API Explorer

A fake-but-convincing REST API documentation and testing tool.
Built with **HTML + Tailwind CSS v4 + Vanilla JS + Chart.js**.

---

## File Structure

```
queryvault/
├── index.html      # Full app — 4 pages rendered via JS (no routing)
├── style.css       # Minimal CSS: method badges, active states, JSON colours
├── script.js       # Page switcher, fake API tester, Chart.js, rate limit animation
├── input.css       # Tailwind v4 entry point (@import "tailwindcss" + @theme config)
├── package.json    # npm scripts — dev, build, serve on port 3010
├── README.md       # This file
└── dist/
    └── output.css  # ← generated by Tailwind CLI (don't edit)
```

---

## Tailwind v4 — Key Differences from v3

| v3                                       | v4                                                |
| ---------------------------------------- | ------------------------------------------------- |
| `tailwind.config.js`                     | Configuration is in `input.css` using `@theme {}` |
| `@tailwind base/components/utilities`    | Single `@import "tailwindcss"`                    |
| `content: ['./index.html']` in JS config | Auto-detected by `@tailwindcss/cli`               |
| `npm install -D tailwindcss`             | `npm install -D tailwindcss @tailwindcss/cli`     |

## Setup & Local Preview

```bash
# 1. Install dependencies
npm install

# 2. Run Tailwind watcher + local server on port 3010 simultaneously
npm run dev
```

Then open **http://localhost:3010** in your browser.

The `dev` script runs two things in parallel via `concurrently`:

- `tailwindcss --watch` — recompiles `dist/output.css` on every save
- `serve . -l 3010` — serves the project on port 3010

> Port 3010 is used here because 3000 is reserved for DevFolio.

---

## Manual commands (if you prefer separate terminals)

**Terminal 1 — Tailwind watcher:**

```bash
npx @tailwindcss/cli -i ./input.css -o ./dist/output.css --watch
```

**Terminal 2 — local server:**

```bash
npx serve . -l 3010
```

## Customisation

### Add a new endpoint

In `script.js`, add a new entry to the `ENDPOINTS` object:

```js
'get-products': {
  method: 'GET', methodClass: 'text-green-400',
  path: '/v1/products',
  description: 'List all products.',
  defaultUrl: 'https://api.queryvault.dev/v1/products',
  params: [ ... ],
  fakeResponse: { status: 200, statusText: '200 OK', body: `{ ... }` },
},
```

Then add a button in `index.html`:

```html
<button data-ep="get-products" class="eptab ...">
  <span class="text-green-400 mr-1">GET</span>/products
</button>
```

### Change the port

In `package.json`, change `-l 3010` to your desired port in both the `dev` and `serve` scripts.
